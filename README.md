## フィードバックの送り方
ご意見・ご提案を歓迎します。4つの方法をご用意しています。

---

### 🟢 方法A：Google Form（GitHubアカウント不要・最も手軽）

**→ [フィードバックフォームを開く](https://forms.gle/XXXXXXXXXX)**

GitHubに不慣れな方や、匿名でご意見を寄せたい方に適しています。  
フォームでは以下についてご意見をいただけます：
- 章・節の過不足、順序への提案
- 対象読者・難易度との適合性
- 自由記述によるコメント

---

### 🔵 方法B：GitHub Issue（定型的な指摘・提案）

[Issues タブ](../../issues/new/choose) から Issue を作成してください。  
以下のテンプレートを用意しています（`.github/ISSUE_TEMPLATE/` 参照）：

| テンプレート | 用途 |
|---|---|
| `📌 構成への提案` | 章・節の追加・削除・順序変更の提案 |
| `❓ 質問` | 構成の意図や執筆方針への質問 |
| `🐛 誤記・不明点の指摘` | 表記ミス、用語の不統一など |

---

### 🟡 方法C：GitHub Discussions（自由な議論・意見交換）

[Discussions タブ](../../discussions) をご利用ください。  
章をまたぐ大きなテーマや、教科書の方向性についての議論に適しています。

カテゴリ例：
- **General** — 全般的な意見・感想
- **内容の深さ・範囲** — 扱うトピックの選定について
- **教育的観点** — 授業での使いやすさ、演習の充実度など

---

### 🔴 方法D：プルリクエスト（具体的な修正案を直接提出）

誤字・脱字の修正や、章節構成の代替案を具体的なテキストで提案したい場合は、Pull Request を歓迎します。

**手順：**
1. 本リポジトリを Fork する
2. 修正・加筆を行い、変更内容を commit する
3. `main` ブランチに向けて Pull Request を作成する
4. PR 本文に「何を・なぜ変更したか」を記載する

> ⚠️ 大きな構成変更の提案は、プルリクエストの前に Issue や Discussions で議論することを推奨します。

---

## フィードバックの取り扱いについて

- いただいたご意見はすべて拝読しますが、個別には返信・回答ができない場合があります。
- 内容の採否は執筆者が判断します。採用・不採用の理由を必ずしも説明できるとは限りません。
- 受領した意見の概要は Discussions にて定期的に報告します。
- Google Form の回答は、匿名化・集計のうえ共有することがあります。
- 氏名・所属等の個人情報は収集しません。回答上、個人情報や機密情報を記載しないように十分ご注意ください。
